# Incident Response Plan (TMM)

Version: 1.0  
Owner: Security Lead  
Review cadence: Quarterly

## 1. Severity levels

- Sev1: active data breach or critical compromise.
- Sev2: major outage or high-risk security event.
- Sev3: contained security defect or suspicious activity.
- Sev4: low-risk event requiring tracking.

## 2. Response phases

1. Detect and triage
2. Contain
3. Eradicate and recover
4. Communicate and document
5. Post-incident review

## 3. Initial response SLA

- Sev1: acknowledge within 15 minutes
- Sev2: acknowledge within 30 minutes
- Sev3/Sev4: acknowledge within 4 business hours

## 4. Communication

- Internal incident channel + incident commander assigned.
- External communications coordinated by executive owner.
- Document timeline, impacted systems, and customer impact.

## 5. Required artifacts

- Incident ticket
- Timeline log
- Root cause analysis
- Corrective and preventive actions with owners and due dates
