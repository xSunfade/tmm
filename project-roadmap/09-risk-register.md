# Risk Register

Ranked risks for the post-audit development phases. Each has a mitigation already embedded in the plan (cross-referenced) plus a trigger that should cause re-evaluation. Severity = impact × likelihood, post-audit but pre-mitigation.

| # | Risk | Sev | Mitigation in plan | Watch trigger |
|---|---|---|---|---|
| R-1 | **Domain-model rework (D4) destabilizes the simulation core.** The position model + checkpoint semantics touch the product's central promise; a numeric regression would be invisible to casual testing and fatal to trust. | High | Property-test suite (conservation, zero rounding loss) runs on every PR throughout Phase 3; golden fixtures migrated *first*, then refactor; engine contract frozen (ADR-2 boundaries); Simulation role in `tmm-workforce/` owns sign-off | Any golden-fixture change without an explained, dedicated commit |
| R-2 | **Scope creep in the domain model.** D4 explicitly warns against overbuilding (dividends, tax lots, rebalancing…), but the temptation is structural. | High | v1 scope list is written into ADR-2; review gate rejects domain-model PRs adding un-scoped concepts; extension points documented instead of implemented | Phase 3 exceeding 3 weeks |
| R-3 | **Billing bugs at launch** (TMM+ is live at MVP per D2, so PAY-x gaps are launch-critical, not fast-follow). Wrong entitlements or past_due limbo destroy trust faster than feature bugs. | High | Entitlement layer is table-driven + unit-tested over the full status matrix (4.1); test clocks (4.4); founder dogfoods a real billing cycle before Gate D; reconciliation job post-MVP (PAY-7) | Any manual `plan_tier` fix-up in prod |
| R-4 | **Plaid webhook forgery before SEC-1 lands** or misconfigured verification after. Unauthenticated revocation = data-deletion vector. | High | SEC-1 in Phase 4.9 with accept/reject tests; production traffic gated on it (Gate C blocker); forged-webhook test in the lifecycle matrix | Any webhook 200 without verification in prod logs |
| R-5 | **Migration discipline fails again** (hand-applied SQL recurs), causing dev/staging/prod drift exactly like the current repo↔live drift. | Medium-High | CLI-managed baseline (Phase 2.1); staging is *born* from the migration set (proves reproducibility); rule: no SQL outside migrations; CI shadow-apply | `list_migrations` on any project disagreeing with the repo |
| R-6 | **The plan-schema v2→v3 migration corrupts or loses user plans.** | Medium-High | Server persistence + 20 revisions land *before* the migration (Phase 2 before 3); automatic `pre_migration` revision; fixture tests for all historical shapes; corrupt-parse recovery (1.4) keeps raw blobs | Any migration-failure report lacking a recoverable blob |
| R-7 | **Free-signup spike overruns Supabase free tier / costs** before the soft cap works. | Medium | D1 soft-cap + overflow waitlist (4.7); usage alerts at 70/90% (5.6); Supabase Pro upgrade is a $25 lever, pre-approved in the cost plan | Signup rate > plan; DB size > 70% |
| R-8 | **$5/mo price ships by inertia** and TMM+ users are unprofitable. Transactions bills **$0.30 per connected _account_/month** (not per item). | Low (largely mitigated) | Item cap dropped to **TMM+ 3 / Pro 6** to bound worst case; floor recomputed (TMM+ ~$8.50 typical / ~$12 worst; Pro ~$13.50 / ~$19); **prices set: TMM+ $12/$120, Pro $25/$250** — clear ≥30% margin even at worst case (monthly). Gate C checklist includes "floor analysis signed off"; re-verify vs first real Plaid invoice before Gate D | Checkout enabled with the placeholder $5 price; accounts-per-item running higher than assumed; annual worst-case margin (~22%) if that cohort grows |
| R-9 | **Always-on backend migration slips** and production launches on Vercel serverless, silently breaking the sync worker and schedulers. | Medium | Phase 5.3 is a Gate C blocker; worker/scheduler liveness is an explicit smoke check in the deploy pipeline; `08-infrastructure-inventory.md` documents why serverless is invalid | Stuck `plaid_sync_jobs` accumulating in prod |
| R-10 | **Solo-founder bus factor / alert fatigue.** All alerts go to one inbox (D25); a missed billing or security alert has no backstop. | Medium | Alert routing tested (5.6); daily-check ritual in the ops runbook; post-MVP: reconciliation jobs reduce reliance on alert vigilance; workforce review gates catch what monitoring misses | Any alert unacknowledged > 48 h |
| R-11 | **Google Sheets users misread the demotion** (D5) and lose edits made directly in Sheets expecting them to sync. | Low-Med | Explicit Export/Import UI copy (2.7); import always snapshots first (2.5); docs/FAQ language | Support emails about "my sheet changes disappeared" |
| R-12 | **Google OAuth verification delays** block Sheets for external users longer than expected. | Low | D21 makes it non-blocking (beta label, internal users); verification submitted early in parallel | Verification still pending at Gate D |
| R-13 | **Legacy engine deletion breaks a hidden consumer** (tests or UI reading legacy-only fields). | Low | Test migration precedes deletion (3.5); grep gate for imports; one revertible PR | Build/tests red after deletion PR |
| R-14 | **MCP/agent tooling writes to the wrong environment** (e.g., an agent runs SQL against prod). | Low now (no prod project yet), High later | Operating rule: MCP writes = dev only; staging via migration PRs; prod via pipeline only (`tmm-workforce/operating-rules.md`); prod project uses separate credentials not exposed to the workspace | Any non-pipeline write in prod audit logs |

## Standing risk-review ritual

At each phase boundary (and at Gates A–D), the Release Manager role walks this table: re-score, close mitigated rows, add new rows. The register lives here; history via git.
