# Access Onboarding and Offboarding Procedure (TMM)

Version: 1.0 
Owner: Engineering Manager  
Review cadence: Semiannual

## 1. Onboarding

Before granting access:

1. Confirm role and required systems.
2. Grant minimum role needed.
3. Verify MFA setup is complete.
4. Provide security handling requirements (secrets, data, incident reporting).

Record:

- Date, approver, systems granted, role level.

## 2. Role change

1. Re-evaluate required permissions.
2. Remove obsolete access.
3. Reconfirm MFA and ownership.

## 3. Offboarding

Within same business day:

1. Revoke all access to critical systems.
2. Rotate shared secrets if exposure risk exists.
3. Transfer ownership of automation/secrets/alerts.
4. Document completion.

## 4. Quarterly audit

- Compare HR/team roster with system access lists.
- Remove orphaned/stale accounts.
- Archive audit evidence.
